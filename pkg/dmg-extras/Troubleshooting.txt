Deli DL-888D PRO — Troubleshooting Decision Tree
==================================================

You're still seeing "4 empty labels + 5/6 half-half" or similar bad
feeding. Follow these steps in order — do NOT skip ahead.

Step 1 — Install v1.4.0
  Double-click "Install Deli DL-888D PRO Driver.pkg".
  Installer creates BOTH queues automatically:
    Deli_DL888D_PRO      (normal, filter + PPD)
    Deli_DL888D_PRO_Raw  (raw TSPL/ZPL passthrough)

Step 2 — Is the printer calibrated to its label stock?
  Double-click "03 Calibrate.command".
  Printer should feed 2-3 blank labels and stop.
  If it does NOT feed at all, do the HARDWARE calibration:
    - Power OFF
    - Hold FEED button and power ON (keep holding)
    - Release when the status LED blinks red twice
    - Printer auto-feeds 2-3 labels
    - Power cycle

Step 3 — What protocol does your printer speak?
  a. Double-click "01 Test TSPL.command".
     Expected: ONE 60x30 mm label with a border and three lines of
     text: "TSPL TEST / 60 x 30 mm / Deli DL-888D PRO".
     → If this works, TSPL is correct. Go to Step 4.
     → If nothing happens or blanks feed, go to (b).

  b. Double-click "02 Test ZPL.command".
     Expected: ONE 60x30 mm label with three lines of text.
     → If this works, your printer is a ZPL device, NOT TSPL.
       Send the diagnose report (Step 5) and I'll rewrite the
       filter to emit ZPL.
     → If nothing works, go to (c).

  c. Run "04 Diagnose.command", send me the report on the Desktop.

Step 4 — TSPL test works, but normal printing from apps doesn't
  This means the rastertotspl filter is still wrong. Run
  "04 Diagnose.command" — the report includes a dry-run of the
  filter against a synthetic 60x30 mm input. Send me:
    - The report file from the Desktop
    - Whether the dry-run TSPL starts with `SIZE 60 mm,30 mm` or
      something else (locale bug would show `SIZE 60,0 mm,30,0 mm`)

Step 5 — Diagnose report
  Double-click "04 Diagnose.command". It writes
      ~/Desktop/deli-diagnose-<date>-<time>.txt
  and opens it. Copy the entire content and paste it in the issue
  tracker or to me directly. The report includes:
    - Driver version actually installed
    - PPD file info
    - Both queue states
    - USB device info
    - Last 50 CUPS error log lines
    - A dry-run of rastertotspl against a synthetic input
      (first 400 bytes of TSPL output, hex + text)
  This lets me see exactly what your system is doing without having
  physical access to the printer.

Common symptom → cause map
--------------------------
4 blank + 5/6 half-half    → Printer interprets SIZE as bigger than
                             60x30, feeding ~5 labels per job.
                             Possible causes:
                               - Printer needs gap calibration
                                 (Step 2)
                               - Printer doesn't speak TSPL
                                 (Step 3b — try ZPL)
                               - Filter is emitting wrong SIZE
                                 (Step 5 — diagnose report)

Nothing prints at all      → USB connection / backend issue.
                             Check `lpinfo -v` shows the printer.
                             Check lpstat -p shows queue is idle.
                             Run "04 Diagnose.command".

Garbage characters print   → Wrong protocol. Try 02 Test ZPL.command.

Correct label but shifted  → Adjust *Density and *Speed in PPD via
                             Printer Options, or add REFERENCE X,Y
                             values.
