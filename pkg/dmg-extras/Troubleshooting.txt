Deli DL-888D PRO macOS Driver — Troubleshooting
=================================================

STEP 1: Install the driver
  Double-click `Install Deli DL-888D PRO Driver.pkg` in this DMG.
  If Gatekeeper blocks it: right-click → Open → Open Anyway
  (or Settings → Privacy & Security → "Open Anyway").

  The installer creates two CUPS queues automatically:
    - Deli_DL888D_PRO      → normal queue (filter + PPD)
    - Deli_DL888D_PRO_Raw  → raw TSPL queue for diagnostics

STEP 2: Run the raw test print
  Double-click `Test Print.command`.
  You should see ONE 60 x 30 mm label with a border and the words
  "DELI TEST / 60 x 30 mm / raw TSPL OK".

  • If this works → the USB path + printer + TSPL parser are all fine.
    Proceed to step 3 (normal printing).

  • If labels come out blank or content spans multiple labels →
    the printer needs physical gap calibration:
      1. Power OFF the printer.
      2. Hold the FEED button while powering it ON.
      3. Keep holding until the LED blinks red twice.
      4. Release. The printer feeds 2-3 labels to measure the gap.
      5. Power cycle. Re-run the Test Print.

STEP 3: Normal printing from an app
  Open any app (Preview, Pages, TextEdit, etc.).
  In the print dialog:
    - Printer: Deli_DL888D_PRO
    - Paper size: pick one of the 28 presets (e.g. "60 x 30 mm")
    - Print.

  If the paper sizes don't appear in the dropdown:
    - Open System Settings → Printers & Scanners
    - Select Deli_DL888D_PRO → Options & Supplies → Options
    - Verify Resolution = 300 dpi and Density = 8

STEP 4: Diagnosing bad output
  Run in Terminal:
      log show --style compact --last 2m \
        --predicate 'process == "cupsd"' | grep DEBUG

  You should see a line like:
      DEBUG: page=1 PageSize=[170,85]pt->60x30mm raster=708x354@300x300dpi ...

  The PageSize in points must match your selected paper (60x30 = [170,85]).
  If it doesn't, the app sent the wrong paper size — re-check the
  print dialog.

STEP 5: Uninstall
  Open Terminal and run:
      sudo lpadmin -x Deli_DL888D_PRO
      sudo lpadmin -x Deli_DL888D_PRO_Raw
      sudo rm -f /usr/libexec/cups/filter/rastertotspl
      sudo rm -f /Library/Printers/PPDs/Contents/Resources/deli-dl888d-pro.ppd
      sudo launchctl kickstart -k system/org.cups.cupsd

Log files:
  - Install log:  /var/log/install.log   (search for "[deli-driver]")
  - CUPS log:     /var/log/cups/error_log
  - unified log:  log show --last 10m --predicate 'process == "cupsd"'
